# Техническое описание проекта GamerGate

## 1. Функциональные возможности сайта
Сайт представляет собой веб-приложение для интернет-магазина игровых ключей со следующими ключевыми функциями:
* **Каталог товаров:** Просмотр списка доступных игр с отображением их названий и цен.
* **Система корзины:** Добавление товаров в корзину, динамическое изменение количества позиций (x1, x2 и т.д.) и удаление товаров.
* **Оформление заказа:** Страница чекаута с выбором способа оплаты (привязанная карта, новая карта, баланс кошелька) и вводом контактных данных (Email).
* **Пользовательский интерфейс:** Личный кабинет пользователя с отображением баланса, поиск по сайту и навигация по категориям (Список желаемого, Профиль).

## 2. Перечень технологий и библиотек
Проект реализован с использованием базового стека веб-технологий:
* **Язык разметки:** HTML5 (использование семантических тегов `<header>`, `<footer>`, `<main>`, `<section>`).
* **Стилизация:** CSS3 (использование Flexbox для адаптивной верстки, медиа-запросы для поддержки различных разрешений экрана).
* **Программирование:** JavaScript (ванильный JS для управления состоянием корзины, манипуляций с DOM и обработки событий кнопок).
* **Инструменты:** Браузерные инструменты разработчика для отладки и тестирования верстки.

## 3. Обоснование принятых дизайн-решений
* **Цветовая палитра:** Использование темной темы (черный фон, серые акценты) обусловлено спецификой игровой индустрии, что снижает нагрузку на зрение и акцентирует внимание на визуальном контенте (обложках игр).
* **Flexbox-верстка:** Выбрана из-за гибкости при создании адаптивных карточек товаров. Это позволяет элементам внутри карточек автоматически подстраиваться под ширину родительского контейнера.
* **Центрирование элементов:** Решение центрировать цену и контент внутри карточек корзины принято для визуальной симметрии и упрощения восприятия информации пользователем при покупке.
* **Разделение на панели:** Вынос выбора способа оплаты и формы Email в отдельные панели (`checkout-panel`) помогает структурировать процесс оформления заказа, уменьшая когнитивную нагрузку на пользователя.
